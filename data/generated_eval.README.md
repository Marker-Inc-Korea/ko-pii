# generated_eval.jsonl — 합성 한국어 PII 평가셋

행정/서식체 한국어 문서에 대한 **다중 시스템 PII 검출 벤치마크** 데이터셋.
모든 인물·번호는 **LLM 생성 합성값**으로 실제 개인정보가 아니다.

## 개요

| 항목 | 값 |
|---|---|
| 문서 수 | 540 |
| gold PII span | 3,635 |
| 라벨 종류 | 26 |
| 중복 | 0 |
| 빈-gold 문서(오탐 테스트) | 18 |
| 평균 문서 길이 | ~309자 |

## 형식

JSONL, 한 줄당 한 문서:

```json
{"text": "...본문...", "pii": [{"type": "PERSON", "text": "홍길동"}, ...], "domain": "..."}
```

gold의 모든 `text`는 본문에 **글자 그대로** 등장한다(부분집합 매칭 채점 호환).

## 생성 + 검증

1. **생성** — LLM으로 행정/공문서·민원·계약·의료·인사 등 다양한 도메인의 현실적 문서를
   중복 없이 생성, 8대 제품군/26개 PII 라벨을 폭넓게 커버.
2. **자동 정제** — 중복 제거 + "gold가 본문에 존재" 검증 + 16자리 카드 Luhn 보정.
3. **gold 검증 (2단계)** —
   - 자동 형식검증(타입별 정규식 + 본문 존재 확인)
   - LLM 적대적 감사(20개 에이전트가 분담, 오라벨/누락/오탐/span오류 검사)
   - 결과: 45건 실오류 / 3,634 = **약 1.2% (98.8% 정확)**, **37건 수정**(제거 16 + span트림 4 + 누락추가 17).

## 주의 (해석)

gold가 LLM 생성이라 **소프트 속성(POSITION·EDUCATION 등)의 라벨 컨벤션이 다소 느슨**하고,
이 때문에 LLM 기반 검출기에 유리한 편향이 있다. 독립 인간 라벨 벤치(KDPII)와 함께 보는 것을 권장.

## 채점

`ko_pii.eval.kdpii.match_forms_overlap` (position-insensitive 부분집합 매칭),
`person_min_length=3` 동일 적용. 측정 결과는 `docs/BENCHMARK.md` 참조.
